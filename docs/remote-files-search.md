# Поиск запрещенных файлов у пользователей

## Что это

Отдельный набор для удаленного запуска `Everything` на ПК пользователя и последующего поиска файлов через ETP-подключение.

Основная логика:
- portable-папка копируется на удаленный ПК;
- `Everything.exe` стартует там через `PsExec`;
- затем ты подключаешься к ETP-серверу из локального `Everything64.exe`.

## Как запускать

1. Проверь настройки в `Remote Files Search (запуск от имени админа).bat`.
2. Проверь `EverythingPortable/Everything.ini`.
3. Запусти BAT от имени администратора.
4. Введи IP или имя удаленного ПК.
5. Введи логин и пароль администратора.
6. После старта открой локально `Everything64.exe`.
7. Подключись к ETP-серверу на удаленном хосте.

## Что нужно настроить

В BAT-файле:
- `src` — путь к локальной portable-папке;
- `userName` — админская учетка по умолчанию;
- `remoteInstallDir` — путь установки на удаленном ПК.

В `Everything.ini`:
- `etp_server_port`;
- `etp_server_username`;
- `etp_server_password`.

## Особенности

- использует `robocopy` для копирования на `\\HOST\c$`;
- использует `PsExec` для запуска `Everything.exe` на удаленном хосте;
- требует отдельной настройки `Everything.ini` под твою среду.

## Узкие места

- нужны права локального администратора на удаленном ПК;
- должны работать admin shares и SMB;
- firewall не должен блокировать SMB и ETP-порт;
- в репозитории нельзя хранить реальные логины, пароли и чувствительные значения из `Everything.ini`;
- в `EverythingPortable` нежелательно держать рабочую базу `Everything.db` и историю запусков.

## Что я бы держал под контролем

- не коммитить реальные учетные данные;
- не коммитить рабочие артефакты клиента;
- периодически проверять, не попали ли в `Everything.ini` реальные IP, логины или пароли.
