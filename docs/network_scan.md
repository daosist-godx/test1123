# network_scan.ps1

## Что делает

Небольшая PowerShell-тулза для прохода по подсети `/24`.

Скрипт:
- пингует адреса `1..254`;
- пытается узнать имя ПК;
- пытается вытащить текущего пользователя;
- если может, тянет `DisplayName` из AD;
- складывает результат в `TXT` или `CSV`.

## Как запускать

```powershell
powershell -ExecutionPolicy Bypass -File .\network_scan.ps1
```

Дальше по шагам:
1. вводишь подсеть вида `192.168.1` или `10.0.0`;
2. выбираешь формат: `1` для `TXT`, `2` для `CSV`.

## Особенности

- по умолчанию идет по диапазону `1..254`;
- для другой `/24` код трогать не надо, просто вводишь другой префикс;
- CSV пишется с разделителем `;`;
- скрипт интерактивный, не под пайплайн, а под быстрый ручной запуск.

## Узкие места

- WMI-опросы могут отваливаться из-за прав или firewall;
- `Get-ADUser` требует доступного AD-модуля и нормального доступа в домен;
- DNS-имя определяется не всегда;
- на медленной сети тулза может работать небыстро.
